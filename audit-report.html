⚖️ Automation — Pros & Cons
What worked well and what were the limitations of this approach.

✅ Pros

Speed on retestFull 75-item audit reruns in ~3 minutes after fixes are deployed.

ConsistencyEvery test runs identically — no human fatigue, no variation.

Living documentSpreadsheet auto-updates after every run, always reflecting current state.

ScalabilityAdding new tests for new features takes minutes, not hours.

Rich evidenceEach result records exact observed behaviour, WCAG reference, and severity automatically.

❌ Cons

Video content needs manual testingVideos protected by digital rights management (DRM) can't be played by the automation tool — the Media Player section had to be tested by hand.

Re-login required each runThe app session expires after about an hour, so you need to log back in before each test run.

Can't always judge what it seesThe tool can detect that something changed on screen, but it can't always tell whether that change was correct or meaningful — a human eye is still needed for some checks.

3 tests were blocked by existing bugsSome tests couldn't run because bugs in the app prevented reaching the feature being tested — those will need to be re-checked once the bugs are fixed.

💡 Interesting Observations
Unexpected findings and patterns from the audit.
🎯

The EPG is the biggest single failure

The TV Guide grid is reachable but completely non-interactive — it scrolls but nothing can be selected. For a core feature of a TV platform, this means the entire programme guide is effectively decorative for keyboard users.

🔐

DRM is an unexpected automation barrier

TELUS TV+ uses Widevine DRM for all streams. Playwright's Chromium lacks the CDM plugin needed to decrypt them — a known industry-wide limitation. Any streaming platform audit must account for this.

♿

Missing features hurt the users who need them most

The two highest-priority missing Media Player shortcuts — C (captions) and V (described video) — directly affect deaf and blind users. These are the people most likely to rely on keyboard navigation.

🖱️

Right-click menu is a Mac-specific gap

The Context Menu key exists on Windows keyboards but not MacBook. Since TELUS TV+ is primarily Apple-targeted, the entire contextual menu is mouse-only. A ⋯ button on cards would fix all platforms at once.

🔄

Modal failures are systemic, not isolated

The same three modal failures (no focus trap, no Esc, no focus on open) appeared on every modal tested. This is an architectural pattern that needs fixing at the shared modal component level.

✨

Authentication is a bright spot

Despite widespread issues, the login flow is well-implemented — auto-focus, logical Tab order, Enter and Space both work. The only gap is a one-line fix (role="alert" on error messages).
